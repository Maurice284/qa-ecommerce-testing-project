# QA Test Plan – E-commerce Website

## 1. Project Overview

The QA E-commerce Testing Project validates the core functionality of an e-commerce web application similar to Amazon. The purpose of this project is to demonstrate practical QA skills, including test planning, test case creation, bug tracking, and test execution.

## 2. Scope

### In Scope
- User Registration and Login
- Product Search and Filtering
- Product Details Page
- Add to Cart functionality
- Cart Management (update/remove items)
- Checkout Process
- Coupon Code Validation

### Out of Scope
- Performance Testing
- Security Testing
- Third-Party Payment Gateway Validation

## 3. Objectives

The objectives of this testing project are:

- Verify that all key features function as expected
- Identify and document defects
- Ensure a smooth user experience
- Confirm users can successfully complete a purchase

## 4. Testing Types

### Functional Testing
- Verify all features behave according to requirements
- Examples: Login, Add to Cart, Checkout

### UI Testing
- Check the layout, buttons, links, and responsiveness
- Ensure visual elements are correct and aligned

### Regression Testing
- Re-test features after bug fixes
- Ensure previously working features still function

## 5. Test Approach

- Manual testing will be performed
- Test cases will cover all core features
- Bugs will be logged with severity and priority
- Test execution will include Actual Result and Pass/Fail status
- Retesting will be conducted after bug fixes

## 6. Entry Criteria

- The application is accessible and functional
- Test environment is set up and stable
- Test cases are created and ready

## 7. Exit Criteria

- All planned test cases are executed
- Critical and high-severity bugs are resolved
- Test execution results are documented

## 8. Deliverables

- Test Plan Document
- Test Cases Document
- Bug Reports Document
- Test Execution Report

## 9. Roles and Responsibilities

| Role | Responsibility |
|------|----------------|
| QA Tester | Execute test cases, log bugs, report results |
| QA Lead | Review test plan and test cases, monitor testing progress |
| Developer | Fix reported bugs and provide updates |
